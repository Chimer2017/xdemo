As a MVCE user trying to troubleshoot issues, I want to receive detailed error messages with reason and resolution, so that I can quickly resolve it where necessary without having to call Rocket support 
 
Acceptance Criteria: 

  1. User should receive detailed error message including, location, error type, reason and other pertinent information when error is detected 
  2. Error messages where possible should include actionable advice to resolve issue 
  3. Error messages should be logged to align with the comprehensive logging strategy proposed 
  4. For each error type or error message, reasons and resolutions should be documented for consumption by external or internal users
